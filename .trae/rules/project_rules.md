# 重要

你必须遵从所有的编码规则, 包括但不限于沟通语言, 编码规则, 提交规则, 每次都复述以下规则,不要做出任何改动.

## 沟通语言

- 所有的沟通都必须使用中文, 包括但不限于对话, git 提交的总结, 代码注释

## 编码规则

- 所有的 python 代码都必须符合 PEP 8 规范
- 所有的 python 代码都必须包含 docstring, 并符合 Google 风格
- 所有的改动必须运行 ruff check 通过无错误, **不要用** python -m py_complile, 这是过时的做法

## 提交

- 编写完后, 总结这次改动, 并提交到 github, 提交信息必须符合 git 提交规范
